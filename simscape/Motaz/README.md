# Motaz Simscape Models

This folder contains two Simulink/Simscape Electrical models built around the TWIST board power stage:

- `V1_DC_DC.slx`: DC/DC-oriented model with a fixed top-level duty-cycle input.
- `V1_DC_AC.slx`: DC/AC-oriented model that replaces the fixed duty input with a sinusoidal modulation reference.
- `V1_DC_DC.slx.r2022b`: alternate copy of the DC/DC model saved for R2022b compatibility.

No MATLAB setup script is stored in this folder. The models are primarily configured from block parameters and top-level controls inside each `.slx` file.

## Files

- `V1_DC_DC.slx`: main DC/DC model
- `V1_DC_AC.slx`: main DC/AC model
- `V1_DC_DC.slx.r2022b`: compatibility copy of the DC/DC model
- `images/`: screenshots of the top-level architecture and main subsystems

## Model overview

The screenshots in `images/` show a common architecture shared by both models:

- a control subsystem that generates the gate commands for two legs
- a measurement subsystem that samples currents and voltages
- a TWIST board circuit model that represents the power stage

![Top-level model overview](images/overview.png)

At the top level, the models expose:

- gate-control inputs for both legs
- switch commands for optional capacitor connections
- a measurement scope with six outputs: `I1`, `I2`, `V1`, `V2`, `VH`, and `IH`

The main difference between the two model files is the modulation source:

- `V1_DC_DC.slx` uses a fixed duty-cycle command (`SET DutyCycle`, set to `0.5` in the current model).
- `V1_DC_AC.slx` uses a sinusoidal duty reference for inverter-style operation. The current top-level source is a `50 Hz` sine wave with `0.5` bias and `0.45` amplitude.

## Control subsystem

The control section is organized as two mirrored leg controllers. Each leg receives:

- a duty-cycle input
- an activation input

and produces:

- one high-side gate signal
- one low-side gate signal
- one carrier signal used elsewhere in the model

![Control system overview](images/control_system_overview.png)

Inside each leg controller, the PWM stage uses a 2-level PWM generator, enable logic, dead-time insertion, and complementary gate outputs:

![PWM timer model](images/PWM_timer_model.png)

This makes the control path easy to inspect when validating:

- gate timing
- complementary switching behavior
- carrier-based acquisition timing

## Measurement subsystem

The measurement section exposes six signals:

- `i1Meas`
- `i2Meas`
- `v1Meas`
- `v2Meas`
- `vHMeas`
- `iHMeas`

![Measurement subsystem overview](images/measurements_overview.png)

Each channel is sampled with the carrier signal, as shown by the single-channel acquisition structure below:

![Single measurement overview](images/single_measurement_overview.png)

This arrangement is useful when comparing power-stage behavior with the switching instants generated by the controller.

## TWIST board circuit model

The power stage combines the low-side and high-side sections of the TWIST board with the switching devices, passive components, and sensing points used by the measurement subsystem.

![TWIST board circuit model](images/Twist_ circuit.png)

From the diagram, the power-stage model includes:

- two controlled legs driven by `Q1` to `Q4`
- low-side capacitor and feeder-related circuitry
- current and voltage sensing points routed to the measurement subsystem
- high-side output terminals exposed as `VHigh+` and `VHigh-`

## Which model to start with

- Use `V1_DC_DC.slx` if you want to validate fixed-duty switching behavior, inspect gate timing, or study the board as a DC/DC conversion stage.
- Use `V1_DC_AC.slx` if you want to study sinusoidal modulation and inverter-like DC/AC behavior on the same underlying TWIST board structure.

## Requirements

To open and run these models you need:

- MATLAB
- Simulink
- Simscape Electrical
- Specialized Power Systems support (`powergui` is used in the models)

## Running a model

1. Open either `V1_DC_DC.slx` or `V1_DC_AC.slx` in MATLAB/Simulink.
2. Review the top-level controls and switch states before starting the simulation.
3. Run the model.
4. Inspect the built-in scope outputs: `I1`, `I2`, `V1`, `V2`, `VH`, and `IH`.

## Notes

- `V1_DC_DC.slx.r2022b` appears to be the same DC/DC model stored as an R2022b-compatible copy.
- The screenshots in `images/` are a good guide for navigating the models before editing internal subsystems.
